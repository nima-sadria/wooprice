| `a2_advisory_insights` | id, session_id (FK RESTRICT), category, severity, confidence, summary, explanation, evidence, related_object_type, related_object_id, recommendation_trace, generated_at |

EXPLANATION, RISK_SUMMARY, ANOMALY, STALE_PRICE, REVIEW_PRIORITY, RULE_RECOMMENDATION

### Insight severities

INFO, LOW, MEDIUM, HIGH, CRITICAL

### Advisory output contract

The only output type is `AdvisoryInsight`. No executable domain objects are produced.
Forbidden outputs (never produced): PriceProposal, SafetyResult, ChangeSet, DryRunResult,
ExecutionPlan, Schedule, ApplyCommand.

## Known Risks

- **No real AI API connected**: AdvisoryService uses rule-based classifiers only.
  Advisory quality is limited compared to a real LLM backend. Wiring in a real AI API
  (future phase) will require verification that the new backend respects the output
  contract and does not produce executable objects.
- **Rule-based confidence scores are estimates**: confidence values (e.g., 0.85 for HIGH
  anomaly) are fixed by threshold logic, not derived from probabilistic models.
- **Evidence is stored as JSON string**: SQLite-compatible; no native JSON type. Any
  migration to PostgreSQL must verify JSON round-trip fidelity.
- **No REST endpoints**: A2.9 is a service/repository layer only. REST API surface
  belongs to a future phase.
