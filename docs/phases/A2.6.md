# A2.6 — Dry Run Engine

## Phase

A2.6

## Name

Dry Run Engine

## Status

READY FOR OWNER REVIEW

## Mission

Execute full validation of an approved Change Set without writing to any destination system. The Dry Run Engine produces a structured Dry Run Report and binds seller confirmation to the immutable Change Set digest. Any change to the underlying data, rules, policies, scope, or channel after seller confirmation invalidates that confirmation and requires a new Dry Run.

---

## Scope

- Dry Run execution engine (validates a Change Set in full without writing to any destination)
- `DryRunReport` model (structured output: scope, item count, safety summary, validation result, digest)
- Change Set digest binding (seller confirmation bound to the exact digest of the Change Set)
- Seller confirmation model (records who confirmed, when, against which digest)
- Confirmation invalidation triggers (conditions that invalidate a stored seller confirmation)
- Read-only validation enforcement (the Dry Run Engine must never write to WooCommerce or any channel)
- Dry Run result states: `PASSED` / `WARNINGS` / `BLOCKED`
- Dry Run repository (persistence of Dry Run reports and confirmations)

## Architecture Rules

- Seller confirmation is bound to the immutable Change Set digest. The confirmation is only valid for the exact digest that was confirmed.
- Any change to any of the following must invalidate seller confirmation and require a new Dry Run:
  - Source snapshot (the underlying data changed)
  - Transformation rules (rule versions changed)
  - Safety policies (policy thresholds or modes changed)
  - Scope (products added or removed from the Change Set)
  - Destination channel (target channel changed)
  - Proposal values (any proposed price changed)
- The Dry Run Engine must never write to WooCommerce or any external destination; all operations are read-only.
- A `BLOCKED` Dry Run result must prevent the Execution Engine from proceeding; only `PASSED` and `WARNINGS` results permit execution.
- Dry Run confirmation must record the authorizing user, the confirmed digest, the timestamp, and the Dry Run Report ID.
