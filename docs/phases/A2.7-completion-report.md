## Known Risks

- **DummyAdapter only**: A2.7 does not write to any real destination. When a real adapter is wired in (future phase), the freshness verification and idempotency contracts must be verified against that adapter's behavior.
- **Single batch per execution**: All items are placed in batch_number=1. Multi-batch execution is a future concern.
- **No automatic recovery**: Stale RUNNING detection is implemented but recovery (mark stale as FAILED/BLOCKED) requires a future scheduler or operator action.
