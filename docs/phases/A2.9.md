# A2.9 — AI Foundation

## Phase

A2.9

## Name

AI Foundation

## Status

READY FOR OWNER REVIEW

## Mission

Provide advisory intelligence that assists pricing decisions while remaining completely
outside the Trusted Execution Path. The AI Foundation generates signals, warnings,
and recommendations for human review only. It has no authority to apply prices,
override safety policies, bypass seller confirmation, or trigger any execution.

**A2.9 scope constraint**: No real AI API connections (no OpenAI, no Anthropic).
No existing Workspace, Apply, Dry Run, or Safety Engine logic modified. Advisory
output only — the only output type is AdvisoryInsight.

---

## Trusted Execution Path

The following pipeline is immutable. A2.9 must not modify, replace, intercept,
or bypass any component:

```
Source → Transformation Rule Engine → Safety Policy Engine → Change Set Engine
       → Dry Run Engine → Seller Confirmation → Execution Engine → Scheduling Engine
```

AI is advisory only. It sits alongside this path but never participates in it.

## Architecture Rules

- A2.9 is advisory only. All output is AdvisoryInsight. No executable domain objects
  (PriceProposal, ChangeSet, DryRunResult, ExecutionPlan, Schedule, ApplyCommand)
  are ever produced.
- Isolation is one-way: AI may read prior-phase outputs. Prior phases (Rule Engine,
  Safety Engine, Change Set Engine, Dry Run Engine, Execution Engine, Scheduling Engine)
  must never import from `app.a2.ai`.
- subject_id and related_object_id in advisory models are plain strings — no FK to
  prior-phase tables (phase independence).
- All insights are immutable once created. No update_insight operation exists.
- All sessions must be archived before insights in the session can be considered final.
- Every AdvisoryInsight includes: id, session_id, category, severity, confidence,
  summary, explanation, evidence, related_object_type, related_object_id,
  recommendation_trace, generated_at.
- No AI output may suppress or override a Safety Policy BLOCK result.
- No AI output may act as or replace Seller Confirmation.

## Owner Decisions

- **No auto apply.** AI components may never trigger execution.
- **No safety override.** AI components may never bypass or suppress a Safety Policy result.
- **No seller confirmation bypass.** AI components may never act as a seller confirmation.
- **No WooCommerce write.** AI components hold no destination credentials and make no
  destination writes.
- **No autonomous execution.** AI components may not initiate, schedule, or trigger any
  Change Set execution.
- **No rule generation without Owner approval.** AI may not create or publish pricing
  rules without explicit Owner authorization for each specific rule.
- **No autonomous business-policy modification.** AI may not modify business rules.
- **No autonomous safety-policy modification.** AI may not modify safety thresholds or
  policy modes.
- **No real AI API in this phase.** No OpenAI, Anthropic, or other external AI API
  connections until explicitly approved by Owner.
